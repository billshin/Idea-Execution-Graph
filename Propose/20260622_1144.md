# Idea Execution Graph - Express Backend Plan (Current Version)

## Goal

先為目前版本建立一個可用的 Express 後端，重點是把現有前端的資料保存從 localStorage 過渡到 API。
此階段不處理多使用者登入與權限。

## Scope (Phase 1)

- 單一資料空間（無帳號、無登入）
- 專案列表 CRUD
- 專案圖資料（Graph Snapshot）讀寫
- 健康檢查與版本資訊 API
- 前端可切換使用 localStorage 或 API 模式

## Tech Stack

- Runtime: Node.js + Express
- Language: TypeScript
- DB: SQLite (dev) / PostgreSQL (future)
- ORM: Prisma
- Validation: Zod
- Logging: pino
- CORS: cors middleware

## Data Model (No User Yet)

### Project

- id: string (UUID)
- title: string
- subtitle: string
- snapshotJson: JSON
- createdAt: datetime
- updatedAt: datetime

### Notes

- snapshotJson 直接存完整前端 `GraphSnapshot` 結構
- 後續若要多使用者，只需加 `ownerUserId` 欄位與關聯

## API Design (v1)

### Health

- GET `/Idea-Execution-Graph/api/health`
	- response: `{ ok: true, service: "idea-graph-api", version: "v1" }`

### Projects

- GET `/Idea-Execution-Graph/api/projects`
	- 取得所有專案（依 `updatedAt desc`）

- POST `/Idea-Execution-Graph/api/projects`
	- body:
		- title: string
		- subtitle?: string
		- snapshot: GraphSnapshot
	- 建立新專案

- GET `/Idea-Execution-Graph/api/projects/:id`
	- 取得單一專案

- PUT `/Idea-Execution-Graph/api/projects/:id`
	- body:
		- title?: string
		- subtitle?: string
		- snapshot?: GraphSnapshot
	- 更新專案

- DELETE `/Idea-Execution-Graph/api/projects/:id`
	- 刪除專案

## Frontend Integration Strategy

### Current status

- 目前前端使用 Zustand + localStorage
  - `idea-graph-app/src/persistence/projectStorage.ts`
  - `idea-graph-app/src/persistence/storage.ts`
  - `idea-graph-app/src/store/projectStore.ts`
  - `idea-graph-app/src/store/graphStore.ts`

### Migration path

1. 新增 `API_BASE_URL`（例如 `http://localhost:4000`）
2. 建立 `idea-graph-app/src/api/projectsApi.ts`
3. 在 `idea-graph-app/src/store/projectStore.ts` 加一層資料來源切換：
	 - `mode = local | api`
4. 初期預設仍可用 local，開關切到 api 時改呼叫後端

## Non-Goals (for now)

- 使用者登入/註冊
- RBAC/ACL
- 即時協作
- 檔案上傳服務

## Risks and Decisions

- 風險: snapshot JSON 結構改版可能造成舊資料不相容
	- 對策: 在 snapshot 內加入 `schemaVersion`

- 風險: editor 頻繁自動保存造成請求過多
	- 對策: 前端加 debounce (300~800ms)

## Next Step (Recommended)

先落地 M1 + M2，確保 API 穩定後再讓前端切換到 API 模式。
